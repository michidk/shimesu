shimesu installation online

Publish files. Keep the cloud.

shimesu turns static files and directories into stable HTTPS site URLs inside your own AWS account.

A complete publish flow

$ shimesu publish ./pitch.html --yes

deployed  pitch
url       https://pitch.<your-domain>/

Install and operate.

01

Install the CLI

Install the published binary, then keep a source checkout nearby for the optional raw-template commands.

cargo install shimesu
git clone https://github.com/michidk/shimesu.git
02

Issue the certificate

Deploy the certificate stack in us-east-1, then feed its ACM ARN into the regional data-plane stack.

aws cloudformation deploy \
  --template-file cli/templates/certificate.yaml \
  --stack-name shimesu-certificate \
  --parameter-overrides BaseDomain=static.example.com \
  --region us-east-1
03

Create the regional stack

Use the built-in helper for the regional data plane, then keep the installation up to date with the embedded template.

shimesu stack init \
  --base-domain static.example.com \
  --certificate-arn arn:aws:acm:us-east-1:ACCOUNT:certificate/ID
shimesu stack update
04

Publish and manage

A file becomes its site index. A directory keeps its safe relative tree, and every command can return machine-readable JSON.

shimesu status
shimesu publish ./dist --site demo --yes
shimesu site list --json
shimesu site inspect demo --json
shimesu site delete demo --yes

Built for people and agents.

Human output uses readable progress and confirmations. Automation gets stable JSON for successful results plus explicit non-interactive controls.

shimesu site list --json
shimesu publish ./report --site weekly --yes --json
shimesu site delete weekly --yes

Small, private, and idle-friendly.

Private storageS3 blocks public access and accepts reads only through CloudFront OAC.
HTTPS at the edgeCloudFront and ACM serve the base domain and every site subdomain.
Typed metadataDynamoDB stores site identity and the current deployment receipt.
No server fleetThere are no always-on containers, instances, or application servers.
